# FantiaMosaic v1.0

Fantia の **2026-05-25 改定モザイクガイドライン** ([公式](https://spotlight.fantia.jp/news/260519_guideline))
に既存アップロード画像を後追い適合させるための、Windows 用デスクトップアプリ。

> 「対象の原型が視認不可な状態でのモザイク」が必須要件。
> 公式は「最小 4px 等の絶対値では縮小時に原型が視認できる」と明言。
> 本ツールは**短辺基準の相対値**でモザイク強度を決め、複数の検出器で対象領域を抽出し、
> 矩形・ベタ塗りで仕上げる。

## 特徴

- **ガイドライン準拠のモザイクエンジン** — 短辺%基準でタイルサイズ・ブラー半径を決め、縮小耐性を確保
- **4 種類の検出器**を装備、用途に応じて選べる：
  - 格子モザイク検出 (NCC + 重み付き PCA)
  - 棒線隠蔽検出 (Open-Close + 連結成分 + PCA)
  - DL 検出 (YOLO11-OBB、ONNX Runtime)
  - 軸並行検出 (色クラスタリング・旧版)
- **回転矩形のハンドル編集** — 検出後にマウスで位置・サイズ・角度を微調整可能
- **パラメータ自動チューニング** — 正解矩形を描けば座標降下法で最適パラメータを探索 (単一/複数画像)
- **学習可能な DL モデル** — 自分のデータでファインチューニングして精度向上
- **領域別の塗りつぶし色** — 黒・白・グレー・ベージュ・肌色・ピンクから選択、領域ごとに独立
- **セッション保存とマージ** — 大量アノテーション作業を中断・再開・統合
- **並列バッチエクスポート** — CPU 半数で並列処理、進捗バー・中断対応
- **生成 AI 不使用** — タイルモザイク・ガウシアン・ベタ塗りの純画像処理 + 古典的検出 + YOLO 物体検出

---

## ガイドライン解釈

- **タイルサイズ・ブラー半径は画像短辺の % 指定**。絶対 px 下限 (既定 16px) も併用。
- **PixelateThenBlur モード**: タイル化後に弱いブラーを重ね、階段状エッジが縮小時に原型のヒントにならないようにする。
- **適合チェック**: 「短辺 2% 以上・かつ 16px 以上」を満たさないと UI で赤く警告。
- **動画/高解像度向け強プリセット**: 短辺 5% / 下限 24px / 後段ブラー比 0.4。

| プリセット | RelativeStrength | MinimumStrengthPx | 用途 |
|---|---|---|---|
| 標準 | 3% (短辺) | 16px | 静止画 |
| 強 | 5% (短辺) | 24px | 動画・高解像度 |

## 基本ワークフロー

### 1. 画像をロード

- メニュー `ファイル → 画像を開く` または `ファイル → フォルダを開く`
- もしくは画面に**画像/フォルダをドラッグ&ドロップ**
- 「サブフォルダも含む」チェックで再帰スキャン

### 2. モザイクを掛ける領域を指定

**手描き**:
- **左ドラッグ** → 回転対応の矩形領域を作成 (作った直後に選択される)
- **Shift + 左ドラッグ** → 楕円領域を作成
- 矩形は **4 隅ハンドル**でリサイズ、**オレンジ点**で回転、**中央**を掴んで移動

**自動検出**:
- **格子モザイク検出** — タイル状モザイクの再モザイク用
- **棒線隠蔽検出** — 黒い棒線で隠蔽されてる領域を検出
- **DL 検出** — 学習済み ONNX モデルで検出 (要モデル読み込み)
- 一括処理は `一括` メニューから

### 3. モザイク設定

右パネル「モザイク設定」で：
- **モード**: PixelateThenBlur (推奨) / Pixelate / GaussianBlur / SolidFill
- **強度 (短辺%)**: 3-5% 推奨
- **絶対下限 (px)**: 16 以上
- **塗りつぶし色** (SolidFill 時): 黒/白/グレー/ベージュ/肌色/ピンク
- **サムネイル長辺**: アイキャッチ用の縮小版を同時出力したいときに

「ガイドライン適合チェック」が**赤くなったら設定が弱すぎ**。緑なら OK。

### 4. エクスポート

- 「フォルダ選択」で出力先を指定
- 「一括エクスポート実行」で全画像をまとめてモザイク処理
- 並列処理 + 進捗バー + 中断対応

## 検出器の使い分け

| 検出器 | 得意 | 苦手 |
|---|---|---|
| **格子モザイク検出** | タイルモザイクが既に掛かってる画像の再モザイク | 線画/網点漫画 |
| **棒線隠蔽検出** | 黒い太線で隠蔽された画像 | 漫画ハーフトーン (誤検出多発) |
| **DL 検出** | 学習データ次第で何でも | データセット作成の手間 |
| **手動描画** | 確実 | 枚数が多いと時間がかかる |

迷ったら**手動描画**が最終手段として確実。検出器は「下書き」として使い、ハンドル編集で微調整するのが現実的。

## DL 学習フロー（YOLO11-OBB）

少し手間だが、自分のスタイル/作風に合わせた高精度モデルが作れる。

### Step 1: アノテーション

1. アプリで 100 枚以上の代表画像をロード
2. 各画像で**回転矩形でモザイクすべき領域**を描く (ハンドルで位置・サイズ・角度調整)
3. メニュー `ファイル → DLデータセットをエクスポート (YOLO-OBB)` でデータセット出力

途中で `Ctrl+S` で**セッション保存**しておくと作業を中断・再開できる。

### Step 2: Python 環境

```powershell
# CUDA 対応 PyTorch
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124
pip install ultralytics pyyaml
```

### Step 3: 学習

```powershell
python tools/train_yolo_obb.py "<データセットフォルダ>" --epochs 100 --imgsz 1024 --batch 16 --device 0 --model yolo11s-obb.pt
```

| 引数 | 既定 | 説明 |
|---|---|---|
| `--model` | `yolo11n-obb.pt` | `yolo11s-obb.pt` (small) / `yolo11m-obb.pt` (medium) で精度↑ |
| `--epochs` | 100 | 100-200 が無難 |
| `--imgsz` | 640 | 高解像度なら 1024 推奨 |
| `--batch` | 8 | GPU メモリに合わせて |
| `--device` | `0` | `0`=CUDA:0, `cpu`=CPU |

学習中は `runs/obb/train*/` に metrics と画像が書き出される。完了すると `weights/best.onnx` が出来る。

### Step 4: アプリで使う

1. アプリ起動
2. メニュー `DL → モデル(.onnx)を読み込む` で `best.onnx` を選択
3. メニュー `DL → GPU で推論する` にチェック (GPU 推奨)
4. **「DL検出」**ボタン (単一画像) or **「DL検出 (全画像)」**ボタン (バッチ)
5. 進捗バー表示中、中断可能

### 精度の目安

| mAP50 | 評価 |
|---|---|
| 0.5-0.7 | 実用域 |
| 0.7-0.85 | 良好 |
| 0.85+ | 優秀 |

データ量が増えると劇的に良くなる。117 枚で mAP50=0.6、400 枚で 0.85 程度の伸びが期待値。

### セミ自動アノテーション

データ追加時のワークフロー：
1. 既存セッションを `ファイル → セッション読み込み` で復元
2. 新しい画像をロード (D&D で追加)
3. 既存モデルで **「DL検出 (全画像)」** で下書き生成
4. 誤検出だけハンドル編集で修正
5. 全体をデータセットエクスポート → 再学習

## パラメータ自動調整 (古典検出器向け)

棒線隠蔽検出のパラメータを画像に合わせて自動探索：

1. **基準画像で正解矩形を描く** (ハンドルで精密に)
2. **「パラメータ調整(この画像)」** または **「パラメータ調整(複数画像で学習)」** ボタン
3. 座標降下法で 10 個のパラメータを最適化 (180 評価 × サンプル数)
4. 最適値が UI スライダーに自動反映

評価指標は IoU (検出 vs GT)：
- 0.7+ で優秀、0.5+ で実用、0.3 以下はズレ

## セッション

`*.fmsession.json` で全ジョブと領域を JSON 保存。回転矩形の中心・サイズ・角度・色も完全保持。

### マージ機能

メニュー `ファイル → セッション追加読み込み (マージ)`：
- 同じソースパスのジョブがあれば**領域を結合**
- 違うソースパスは**新規追加**
- 別々にアノテーションした作業を後から統合可能

## トラブルシューティング

### 「DL検出失敗っす: Got invalid dimensions」

学習時の `imgsz` とアプリ側の入力サイズ不一致。v1.0 では**モデル読み込み時に入力サイズを自動取得**して合わせるので発生しないはず。古いビルドなら最新に更新。

### GPU で推論されない

v1.0 から **`Microsoft.ML.OnnxRuntime.Gpu` を標準採用**、`DL → GPU で推論する` もデフォルト ON。
それでも GPU が効かない場合：

- CUDA Driver の確認: `nvidia-smi` で GPU と Driver Version が出るか
- **cuDNN が必要**: ONNX Runtime GPU は CUDA + cuDNN を要求する。NVIDIA から cuDNN 8.x/9.x をダウンロードして DLL を PATH に追加
- アプリ側で `DL → GPU で推論する` にチェックが入ってるか確認
- モデル読み込み後のステータスバーに `GPU(CUDA)` と出ているかチェック。`CPU (GPU失敗: ...)` なら CUDA セットアップに問題あり

### 検出が過剰になる/不足する

- 古典検出器: 右パネル「検出パラメータ」のスライダーを調整。または「パラメータ自動調整」で正解矩形に合わせて最適化
- DL 検出: 信頼度しきい値を上げる/下げる

### ビルドが exe ロックで失敗

`FantiaMosaic.exe` がプロセスで動いてないか確認。アプリを閉じてから再ビルド。

### 漫画ハーフトーンで誤検出が大量に出る

`StripeOcclusionDetector` はハーフトーンドット/効果線を「棒線」と誤認しがち。漫画スタイル画像では **DL 検出を使う**か、**手動描画**が推奨。

## 法的注意

**このツールをもってしても、既存の隠ぺい処理を完全に安全な隠ぺいに置き換えられるわけではありません。
DL 処理後には必ず目視で確認してください。**

本ツールは Fantia 公式が示すガイドラインの数値解釈の一例を実装したものです。
最終的に投稿物の適合性を判断するのは Fantia 運営なので、出力結果を必ず自分の目で確認してから投稿してください。

自動検出 (古典検出器・DL ともに) は学習データやパラメータ依存で誤検出・見逃しが起こります。
検出結果は**あくまで作業の下書き**として扱い、最終チェックは人間の責任で行ってください。
